# Restaurant Demand Forecasting

Forecasts daily delivery demand for the family restaurant using Uber Eats and
DoorDash order history, enriched with weather and holiday data, to recommend
staffing levels. (MLT career-prep project, Kelson Lam, July 2026.)

## Project structure

```
data/raw/         <- put your REAL Uber Eats / DoorDash CSV exports here
data/sample/      <- synthetic test data (generated, not real)
data/processed/   <- cleaned tables the scripts produce
scripts/          <- the pipeline, numbered in run order
outputs/          <- charts, forecasts, staffing recommendations
```

## Getting the real data (do this first!)

**Uber Eats:** log in to [Uber Eats Manager](https://merchants.ubereats.com) ->
Reports -> Request report -> "Order History" (or "Payment Details") -> pick the
widest date range available -> download CSV. Save it to `data/raw/` with `uber`
in the filename.

**DoorDash:** log in to the [DoorDash Merchant Portal](https://merchant.doordash.com)
-> Financials (or Reports -> Report Builder) -> order-level report -> widest
date range -> download CSV. Save to `data/raw/` with `doordash` in the filename.

Aim for at least 6-12 months of history; more history = better seasonality
detection. Multiple files per platform are fine (e.g. one per month).

## Setup (already done on this machine)

```
python -m venv .venv
.venv\Scripts\pip install -r requirements.txt
```

Edit `scripts/config.py` to set the restaurant's latitude/longitude and state
(defaults to Knoxville, TN) so the weather and holiday features are correct.

## Running the pipeline

From the `scripts/` folder, using `..\.venv\Scripts\python.exe`:

```
python 01_ingest_clean.py        # clean + merge exports -> daily sales table
python 02_enrich_features.py     # add weather, holidays, calendar, lag features
python 03_eda.py                 # trend/day-of-week/hourly/weather charts
python 04_model.py               # train models, 14-day forecast, staffing plan
```

Or run everything at once: `run_all.bat`

Add `--sample` to `01_ingest_clean.py` to test the pipeline on synthetic data
(generated by `make_sample_data.py`) before the real exports are in.

## Models compared

- **Seasonal naive** (same weekday last week) - the baseline to beat
- **SARIMA** with weekly seasonality
- **XGBoost** on calendar, lag, holiday, and weather features

Evaluated on the last 28 days held out, using MAE and MAPE. The final output is
a 14-day order forecast with each day tiered into busy / normal / slow for
staffing decisions (`outputs/staffing_recommendations.csv`).
